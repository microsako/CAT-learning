# MAAT 重要性模块公式详解

> 逐个拆解 MAAT(ICDM 2020)Importance Module 的式 (10)–(15),配数值例子。式 (11)(12) 前专门补一节 Skip-Gram 基础,零 NLP 背景可读。

**这个模块只回答一个问题**:IWKC(式 5)里的权重 $w_k$——每个知识点多重要——从哪来?它在**考试开始前**用历史作答数据一次性算好,考试中直接查表。

**整体逻辑链**(先记住这个,公式只是逐环的实现):

1. 知识点重不重要,看它名下的**题目有没有代表性**;
2. 有代表性的题 = 和很多其他题相似的题(考了它,等于间接摸了一批相似题,一题顶多题);
3. 要衡量"题与题相似",得先把每道题变成**向量**(空间中的点),距离近 = 相似;
4. 向量从哪来?**从历史学生的作答记录里学**:被学生们"表现得很像"的题,向量就该靠得近。

对应的流水线:**学题目向量(式 10–12)→ 算每道题的密度(式 13–14)→ 知识点权重 = 名下题目密度的平均(式 15)**。

---

## 1. 式 (10):把"一条作答记录"编码成向量

$$x_{ij} = \begin{cases} \mathbb{1}_{|Q|}(j) \oplus \mathbb{1}_{|Q|}(j), & \text{若 } a_{ij} = 1 \text{(答对)} \\ \mathbb{1}_{|Q|}(j) \oplus \mathbf{0}_{|Q|}, & \text{若 } a_{ij} = 0 \text{(答错)} \end{cases}$$

**符号读法**:$\mathbb{1}_{|Q|}(j)$ 是长度为题库大小 $|Q|$、只有第 $j$ 位是 1 的独热向量;$\mathbf{0}_{|Q|}$ 是全零向量;$\oplus$ 是**拼接**(不是加法),拼完长度 $2|Q|$。

**在干什么**:一条记录要同时说清两件事——"做的是哪道题"和"对了没有"。前半段永远在第 $j$ 位标 1(做的是第 $j$ 题);后半段答对时也在第 $j$ 位标 1,答错时全零。**同一道题,答对和答错是两个不同的"词"**——这是后面 word2vec 类比的伏笔。

**代入例子**:题库 $|Q| = 4$,学生做了 $q_2$:

- 答**对**:$x = [\,0,1,0,0 \;|\; 0,1,0,0\,]$(8 维)
- 答**错**:$x = [\,0,1,0,0 \;|\; 0,0,0,0\,]$

## 2. 补课:Skip-Gram 到底是什么?

式 (11)(12) 看不懂,几乎一定是因为它默认读者熟悉 word2vec。这一节从零讲起,讲完再回去看那两个式子,它们就只是"换了符号的抄写"。

### 2.1 目标:把"词"变成向量,且意思相近的词向量要靠近

计算机怎么表示一个词?最朴素的是独热向量:词表 1 万个词,就用 1 万维、只有一位是 1 的向量。它有个致命缺点:**任何两个独热向量都互相垂直、两两等距**,"猫"和"狗"的距离等于"猫"和"微积分"的距离,语义完全体现不出来。

我们想要的是:每个词一个低维稠密向量(比如 100 维),并且**意思相近的词,向量自动靠近**。word2vec(2013)就是做这件事的工具,Skip-Gram 是它的训练方式之一。

### 2.2 核心思想:一个词的意思,由它的"邻居"定义

语言学里有个分布假设(distributional hypothesis):**经常出现在相似上下文里的词,意思相近**。看例子:

- "我家的**猫**在睡觉" / "我家的**狗**在睡觉"
- "喂**猫**吃罐头" / "喂**狗**吃罐头"

"猫"和"狗"本身几乎从不挨着出现,但它们的**邻居**(我家的、睡觉、喂、罐头)高度重合。Skip-Gram 利用这一点,设计了一个"假任务":**给模型句子中间的一个词(中心词),让它预测周围会出现哪些词(上下文词)**。

关键在于:为了预测得准,模型**被迫**给"邻居相似"的词分配相似的向量——"猫"和"狗"都要预测出同一批邻居,向量不靠近就做不到。**词向量是这个预测任务的副产品**;任务本身(预测邻居)训练完就没人关心了,大家要的只是练出来的向量。

### 2.3 预测概率怎么算?点积打分

给中心词 $c$ 和候选词 $o$,Skip-Gram 用两个向量的**点积**打分:点积越大,表示"$o$ 越可能出现在 $c$ 旁边"。原始版本再用 softmax 把整个词表的分数归一化成概率:

$$p(o \mid c) = \frac{\exp(u_o \cdot v_c)}{\sum_{w \in \text{词表}} \exp(u_w \cdot v_c)}$$

这里有个后面要用的细节:**每个词其实有两套向量**——它当中心词时用 $v$,当"被预测的邻居"时用 $u$。两套分开学,训练完通常只保留一套。

### 2.4 负采样:softmax 太贵,改成"真假判别"

上式的分母要对整个词表(几十万词)求和,每步训练都算一遍,太慢。**负采样(Negative Sampling)** 换个提法:不做"从几十万词里挑一个"的多分类,改做几个**二分类**——给一对词,判断它是"真实共现的对"还是"随机拼凑的假对":

- **正样本**:语料里真实挨着出现的 $(c, o)$,要求模型判"真":$\sigma(u_o \cdot v_c) \to 1$,即点积越大越好;
- **负样本**:保持 $c$ 不动,从词表**随机**抽 $N_{neg}$ 个词 $n$(大概率与 $c$ 无关)拼成假对 $(c, n)$,要求模型判"假":$\sigma(-u_n \cdot v_c) \to 1$,等价于点积 $u_n \cdot v_c$ 越小越好。

$\sigma(x) = \frac{1}{1+e^{-x}}$ 是 Sigmoid,把任意实数压到 $(0,1)$,当作"这一对是真的"的概率。

**为什么必须有负样本?** 若只有正样本(只要求真对的点积大),模型有个作弊解:把**所有**词向量调成同一方向、模长拉到无穷大,则任何两词点积都是无穷大、所有 $\sigma$ 都是 1,目标"完美"达成,但向量什么也没学到。负样本提供反向的力("无关的词要推远"),两股力平衡后,空间里的远近才有含义:**共现过的拉近,没共现的推远**。

每一步训练的几何图像:$v_c$ 被朝正样本 $u_o$ 的方向**拉近**一点,同时被朝远离各负样本 $u_n$ 的方向**推开**一点。

### 2.5 从"句子"到"做题记录":Item2Vec 的一步改造

MAAT 论文引用的其实是 **Item2Vec**——把 word2vec 从 NLP 搬到推荐系统的经典做法,改动只有一处:

- 句子里的词有**顺序**,Skip-Gram 只把窗口内(如前后 5 个词)算作邻居;
- 一个用户买过的商品集合**没有顺序**,Item2Vec 干脆规定:**同一集合里任意两个元素都算共现**(相当于窗口无限大)。

MAAT 照搬到教育场景,对照表:

| word2vec | Item2Vec | MAAT |
|---|---|---|
| 一句话 | 一个用户的购物篮 | 一个学生的全部做题记录 |
| 一个词 | 一件商品 | 一条记录(题号 + 对错) |
| 窗口内相邻 = 共现 | 同一购物篮 = 共现 | 同一学生名下 = 共现 |
| 练出词向量 | 练出商品向量 | 练出题目"测试效应"向量 |

补课结束。带着 2.4 的两条("正样本 $\sigma(u_o \cdot v_c)$、负样本 $\sigma(-u_n \cdot v_c)$")回去看式 (11)(12)。

## 3. 式 (11):训练目标——"两两配对,互相预测"

$$\max \frac{1}{|R_k^H|} \sum_{r_{ki} \in R_k^H} \; \sum_{r_{kj} \in R_k^H,\, j \neq i} \log p(r_{kj} \mid r_{ki})$$

$R_k^H$ 是历史学生 $e_k$ 的全部做题记录。按 2.5 的规定,同一学生名下任意两条记录都算共现,所以:**外层求和遍历他的每条记录 $r_{ki}$(充当"中心词"),内层遍历他其余每条记录 $r_{kj}$(充当"邻居"),把每一对的 $\log p$ 加起来**,除以记录条数,越大越好。双重求和只是"两两配对"的书面写法。

**代入例子**:学生 $e_k$ 做过 3 条记录 $A, B, C$,则有序配对共 6 个,目标就是:

$$\frac{1}{3}\Big[\log p(B|A) + \log p(C|A) + \log p(A|B) + \log p(C|B) + \log p(A|C) + \log p(B|C)\Big]$$

所有历史学生的这个量加总,一起最大化。翻译成人话:**训练模型做到"看见一个学生做过记录 A,就能猜出他历史里还有记录 B"**。和 2.2 节同理,猜得准的唯一途径,是给"总出现在同一批学生名下"的记录分配相近的向量——向量是副产品,预测是手段。

## 4. 式 (12):一对记录的"共现概率" = 1 个正样本项 × $N_{neg}$ 个负样本项

$$p(r_{kj} \mid r_{ki}) = \underbrace{\sigma\big((W x_{ki})^T v_j\big)}_{\text{正样本:拉近}} \; \underbrace{\prod_{l=1}^{N_{neg}} \sigma\big(-(W x_{kn_l})^T v_{n_l}\big)}_{N_{neg}\text{ 个负样本:推远}}$$

先对照 2.4 节逐项"认领":

| Skip-Gram 负采样(2.4 节) | 式 (12) |
|---|---|
| 中心词向量 $v_c$ | $W x_{ki}$(当前记录的向量) |
| 邻居词向量 $u_o$ | $v_j$(题 $q_j$ 的嵌入) |
| 正样本项 $\sigma(u_o \cdot v_c)$ | $\sigma\big((W x_{ki})^T v_j\big)$ |
| 负样本项 $\sigma(-u_n \cdot v_c)$ | $\sigma\big(-(W x_{kn_l})^T v_{n_l}\big)$ |

把式 (12) 代回式 (11) 取对数,连乘变连加:$\log \sigma(\text{正}) + \sum_l \log \sigma(-\text{负})$——这正是标准 SGNS 损失。所以式 (12) 并不是一个严格归一化的概率(各项加起来不等于 1),它只是沿用 word2vec 原论文的记法,把"一正多负"的判别目标写成了乘积形式,不必纠结它"是不是真概率"。

还剩三个符号,逐个拆:

**(a) $W x_{ki}$ 在干嘛?——"两套向量"里的第一套。** 2.3 节说过每个"词"有两套向量。这里中心词的向量不是直接查表,而是用矩阵 $W$($d \times 2|Q|$)乘式 (10) 的独热拼接 $x_{ki}$ 得到。因为 $x$ 里只有一两个位置是 1,矩阵乘法实际是**从 $W$ 的 $2|Q|$ 列里挑列相加**:

- 记录 = 答**对**第 $j$ 题:$x$ 的第 $j$ 位和第 $|Q|+j$ 位是 1 → $W x = W$ 第 $j$ 列 $+$ 第 $|Q|+j$ 列;
- 记录 = 答**错**第 $j$ 题:只有第 $j$ 位是 1 → $W x = W$ 第 $j$ 列。

可以把 $W$ 的前 $|Q|$ 列理解为每道题的"基础向量",后 $|Q|$ 列是"答对加成向量":答对/答错同一道题的两种记录**共享基础列、只差一个加成列**——既区分了对错(2.2 节的类比里它们是两个不同的"词",能有不同的邻居分布),又不至于把同一道题的两种结果学成毫无关系的东西。

**(b) $v_j$——"两套向量"里的第二套,也是我们最终要的东西。** 每道题 $q_j$ 配一个 $d$ 维向量 $v_j$,充当"被预测的邻居"角色。注意:$x$(式 10)区分对错,但 $v$ 按**题**编号、不分对错——每道题最终只有一个嵌入。训练完,$W$ 功成身退丢掉,**保留 $v_j$ 作"测试效应嵌入"**(论文记作 $q_j^{TE}$),供式 (13)–(15) 算距离用。

**(c) 负样本 $n_l$**:随机抽 $N_{neg}$ 条别的记录当"假邻居",通过 $\sigma(-\cdot)$ 要求点积小,即把无关记录的向量推远。作用同 2.4 节:没有它,一切向量会坍缩到"全都无穷相似"的作弊解。

> 细心的话会发现一处不严格:按 word2vec 原版,负样本项应是**中心记录**和负样本嵌入的点积 $\sigma\big(-(W x_{ki})^T v_{n_l}\big)$,而论文写成 $W x_{kn_l}$(负样本自己配自己)。一般认为这是论文记法上的笔误/简写,按标准 SGNS 理解即可,不影响任何结论。

**代入例子**:$d = 2$,$N_{neg} = 1$。设当前记录(中心)的向量 $W x_{ki} = (1, 0.5)$,正样本 $v_j = (0.8, 0.4)$,负样本 $v_n = (-0.5, 1)$:

- 正样本点积:$1 \times 0.8 + 0.5 \times 0.4 = 1.0$,$\sigma(1.0) = 0.731$;
- 负样本点积:$1 \times (-0.5) + 0.5 \times 1 = 0$,$\sigma(-0) = 0.5$;
- $p = 0.731 \times 0.5 = 0.366$。

训练就是不断微调 $W$ 和所有 $v$ 让 $p$ 变大:这一步会把 $v_j$ 朝 $(1, 0.5)$ 方向**拉近**一点、把 $v_n$ **推远**一点。

**为什么训练完,"考察效果相似"的题会聚在一起?** 设想两道题 $q_a, q_b$:会做 $q_a$ 的学生大多也会做 $q_b$,它们总是共同出现在同一批学生的记录里、周围的"邻居记录"也高度重合。那么训练中 $v_a$ 和 $v_b$ 会被**同一批**中心向量反复朝同一方向拉——被同一批力拉的两个点,最后自然停在相近的位置(正如"猫"和"狗"因邻居相同而向量靠近)。这种"近"衡量的不是题面文字像不像,而是**学生做起来的表现像不像**——难度、区分度、考察的能力相近——所以叫"测试效应"嵌入。

## 5. 式 (13):两道题的相似度——距离换算成 0 到 1 的分数

$$Sim^{TE}(q_i, q_j) = e^{-\gamma \| q_i^{TE} - q_j^{TE} \|}$$

$\|\cdot\|$ 是欧氏距离(两点间直线距离),$\gamma$ 是控制衰减快慢的正数旋钮。距离 0 → 相似度 1;距离越远,指数衰减到 0。没有更多内容,就是"把距离翻译成相似度"。

**代入例子**($\gamma = 1$):两题嵌入相距 0.3,$Sim = e^{-0.3} = 0.741$;相距 2,$Sim = e^{-2} = 0.135$。

## 6. 式 (14):密度——这道题周围挤不挤?

$$Den^{TE}(q_j) = \frac{1}{K_N} \sum_{q_i \in N(q_j)} Sim^{TE}(q_i, q_j)$$

$N(q_j)$ 是 $q_j$ 在嵌入空间里的 $K_N$ 个**最近邻**。密度 = 和这几个最近邻的**平均相似度**。

直觉:一道题若处在密集题目簇的中心(周围一堆测试效应相似的题),密度高——**考它一道,等于把整簇题间接摸了一遍,代表性强**;孤零零在角落的题,只能代表它自己,密度低。

## 7. 式 (15):知识点重要性——名下题目密度取平均

$$w_k = \frac{1}{\sum_{(q,k) \in G} 1} \sum_{(q,k) \in G} Den^{TE}(q)$$

式子看着绕,其实分母 $\sum 1$ 就是"数一数知识点 $k$ 名下有几道题",整个式子就是**求平均**:$w_k$ = 关联 $k$ 的所有题的密度平均值。名下的题普遍有代表性 → $k$ 是题库的"主干考点" → 权重大,IWKC 里覆盖它更值钱。

## 8. 一个完整的数值例子:从嵌入一路算到 $w_k$

设训练完成后 5 道题的嵌入($d = 2$)如下,取 $\gamma = 1$,$K_N = 2$:

| 题目 | 嵌入坐标 | 关联知识点 |
|---|---|---|
| $q_1$ | $(0, 0)$ | 代数 |
| $q_2$ | $(0.3, 0)$ | 代数 |
| $q_3$ | $(0, 0.4)$ | 代数 |
| $q_4$ | $(3, 3)$ | 几何 |
| $q_5$ | $(3.2, 3)$ | 几何 |

$q_1, q_2, q_3$ 挤成一簇,$q_4, q_5$ 是远处的一小撮。

**算密度(式 13 + 14)**,以 $q_1$ 为例:它的 2 个最近邻是 $q_2$(距离 0.3)和 $q_3$(距离 0.4):

$$Den(q_1) = \frac{e^{-0.3} + e^{-0.4}}{2} = \frac{0.741 + 0.670}{2} = 0.706$$

同法算出全部:

| 题目 | 最近邻(距离) | 密度 |
|---|---|---|
| $q_1$ | $q_2(0.3),\ q_3(0.4)$ | $0.706$ |
| $q_2$ | $q_1(0.3),\ q_3(0.5)$ | $(0.741 + 0.607)/2 = 0.674$ |
| $q_3$ | $q_1(0.4),\ q_2(0.5)$ | $(0.670 + 0.607)/2 = 0.639$ |
| $q_4$ | $q_5(0.2),\ q_1(4.24)$ | $(0.819 + 0.014)/2 = 0.417$ |
| $q_5$ | $q_4(0.2),\ q_1(4.53)$ | $(0.819 + 0.011)/2 = 0.415$ |

注意 $q_4, q_5$:虽然彼此很近(相似度 0.819),但**第二个最近邻已经要跨到远处的簇**(相似度几乎为 0),密度被拉低——小孤岛的代表性天然弱于大陆。

**算权重(式 15)**:

$$w_{\text{代数}} = \frac{0.706 + 0.674 + 0.639}{3} = 0.673, \qquad w_{\text{几何}} = \frac{0.417 + 0.415}{2} = 0.416$$

代数名下的题聚成大簇 → 权重 0.673;几何是边缘小簇 → 权重 0.416。代回 IWKC(式 5),多样性模块就会**优先保证代数的覆盖,而不为几何浪费太多宝贵题量**。

## 9. 总结

六个公式 = 三步流水线:

$$\underbrace{\text{式 (10)–(12)}}_{\text{word2vec 学题目向量}} \;\to\; \underbrace{\text{式 (13)(14)}}_{\text{距离} \to \text{相似度} \to \text{密度}} \;\to\; \underbrace{\text{式 (15)}}_{\text{密度取平均得 } w_k}$$

核心思想一句话:**被学生们"表现得相似"的题会在向量空间里聚簇;处在大簇中心的题最有代表性;名下题目普遍有代表性的知识点,就是值得优先覆盖的"主干考点"。**

> 提醒:本文件夹的开源代码**没有实现**这个模块(所有知识点等权,$w_k \equiv 1$),所以代码里找不到式 (10)–(15) 的对应实现,属正常现象。
